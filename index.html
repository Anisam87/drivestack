The Problem
Vehicle data in India & SEA is broken
Building anything that needs vehicle data means dealing with fragmented OEMs, undocumented APIs, and zero standardization.
🔌

Fragmented Access

Every OEM has its own proprietary protocol. Tata uses REST, Hyundai uses different OAuth, AIS-140 uses raw TCP. Building each connector is a 6-month engineering project.

🔒

No Consent Layer

India's DPDP Act 2023 requires purpose-bound consent for vehicle data. No platform exists to manage this, forcing developers to build consent flows from scratch.

📊

Raw, Messy Data

Tata reports speed in km/h, AIS-140 in knots. Battery SoC ranges differ between OEMs. Developers spend months normalizing instead of shipping product.

API Reference
Clean, RESTful APIs
One integration. Every vehicle. Go from signup to first API call in under 15 minutes.
GET /v1/vehicles/{id} Vehicle metadata
GET /v1/vehicles/{id}/location Real-time GPS
GET /v1/vehicles/{id}/odometer Odometer reading
GET /v1/vehicles/{id}/ev/battery EV battery status
GET /v1/vehicles/{id}/diagnostics DTC codes
POST /v1/vehicles/{id}/lock Lock doors
POST /v1/vehicles/{id}/charge/start Start charging
GET /v1/auth/connect Owner consent
Python cURL
# Install: pip install drivestack
import drivestack
 
client = drivestack.Client(
api_key="ds_live_..."
)
 
# Get vehicle battery status
battery = client.vehicles.battery(
vehicle_id="veh_abc123"
)
 
print(battery.soc) # 0.785
print(battery.range_km) # 186
print(battery.status) # "charging"
 
# Register webhook
client.webhooks.create(
event="charge.complete",
url="https://app.io/hook"
)
